# 2. Versioning & release strategy across repos

- Status: seed
- Date: 2026-06-30
- Raised by: [ADR 0002](../decisions/0002-repo-landscape-poly-repo-principle.md)

## The thought

A deferred decision called for by ADR 0002. With a poly-repo landscape, each repo
versions independently — we need a deliberate versioning and release strategy
(e.g. SemVer per repo plus a pinned platform version set) so the many repos can be
combined into a coherent platform release.

When this matures, promote it to its own ADR.
